Media object :

Flex peut créer des "media objects" divers sans se prendre la tête avec les plus classiques overflow, clearfix ou autres solutions parfois utiles mais souvent pas pratiques. Les emplacements du texte et de l'image peuvent être modifiés facilement sans toucher au html, et il est tout aussi simple d'intégrer un media object dans un autre "parent" que d'en créer un seul

Holy Grail Layout:

À la base, le Holy Grail Layout est une page avec un en-tête, un pied de page et trois colonnes.

Le Holy Grail Layout est un classique des problèmes avec CSS: l'objectif est avoir un centre fluide avec des barres latérales de largeur fixe. La colonne centrale doit être chargée en premier dans le HTML et toutes les colonnes doivent être de la même hauteur.

Flexbox règle ce problème facilement !

Vertical Centering :

Flex permet des alignements bien plus simples que le CSS 'classique', y compris le centrage vertical, habituellement un casse tête sans nom, avec des possibilités qui devraient théoriquement fonctionner, mais finissent pas alourdir le code sans effet notable (voire, dans certains cas, cassent la moitié de la page). Flexbox aligne efficacement et proprement, que l'élément en question ait ou non des éléments siblings:

align-self: flex-start;
justify-content: center;
align-self: flex-end;

Sticky Footer:

Obtenir le footer de la page avec du contenu clairsemé est quelque chose que chaque développeur a essayé au moins une fois dans sa vie. Et pour la plupart c'est mission accomplie ! Pourtant, ces solutions représentent un gros problème si la taille du footer est inconnue.
Flexbox répond parfaitement à ce type de problème. Bien que connu pour la pose horizontale, il fonctionne aussi bien pour la pose verticale. Tout ce que vous avez à faire est d'envelopper les sections verticales dans un conteneur flex et de choisir ceux que vous souhaitez étendre. Ils prendront automatiquement tout l'espace disponible dans leur conteneur.
